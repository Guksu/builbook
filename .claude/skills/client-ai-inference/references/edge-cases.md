# Client AI Inference — 엣지케이스 가이드

브라우저 온디바이스 추론(Transformers.js + Web Worker)에서 실제로 깨지는 지점들. `dev/assignment`(회사 코드, 복사 금지)의 SSE 채팅에서 검증된 교훈을 Worker 맥락으로 번역했다. AI 문답 구현 시 SKILL.md와 함께 읽는다.

## 목차
1. 모델 다운로드 생명주기
2. 토큰 스트리밍 깨짐
3. 취소(중단) 타이밍
4. 런타임 가용성(WebGPU/WASM)
5. 상태 누락 → 멈춘 UI
6. 메모리·성능
7. Next.js/Worker 빌드 함정
8. UX 디테일(자동 스크롤·첫 토큰)
9. assignment 교훈 매핑표

## 1. 모델 다운로드 생명주기
- **게이팅 우회 금지.** 컴포넌트 마운트·SWR 프리페치 등으로 모델이 *암묵적으로* 받아지면 안 된다. `load` 메시지는 오직 사용자 클릭에서만 발생.
- **중복 다운로드 방지.** 사용자가 버튼을 두 번 누르거나, 다운로드 중 컴포넌트가 리마운트돼도 `load`가 두 번 실행되면 안 된다. Worker는 `generator`가 이미 있으면(또는 로딩 Promise가 진행 중이면) 즉시 `ready`만 보낸다. 메인도 상태가 `"downloading"`이면 추가 `load`를 보내지 않는다.
- **진행률 합산.** `progress_callback`은 파일별로 여러 번, 병렬로 온다. 파일명별 최신 loaded/total을 Map에 보관해 `Σloaded / Σtotal`로 단일 퍼센트를 만든다. 단순 마지막 값만 쓰면 퍼센트가 출렁인다.
- **다운로드 중 이탈/탭 종료.** 진행 중 페이지를 떠나면 브라우저가 fetch를 끊는다 — 부분 캐시는 다음에 이어받거나 다시 받는다(Transformers.js가 Cache API로 처리). 우리 쪽은 이탈 시 상태를 깔끔히 정리(스피너 누수 방지)하면 충분하다.
- **캐시 재방문.** 두 번째 방문에선 네트워크 없이 캐시에서 로드되지만 그래도 수 초 걸린다 — "불러오는 중"을 여전히 보여준다(즉시 ready라고 가정 금지).

## 2. 토큰 스트리밍 깨짐
- **빈 응답.** 모델이 즉시 `done`만 주고 토큰이 없을 수 있다. UI는 빈 말풍선을 영원히 남기지 말고 "응답을 생성하지 못했어요" 폴백을 보인다.
- **부분 단어/유니코드 경계.** `TextStreamer`는 토크나이저가 디코딩 가능한 시점에만 조각을 준다(불완전 멀티바이트 문자를 쪼개지 않음). 그래도 조각을 *append*만 하고 중간에서 자르거나 trim하지 않는다 — 공백·줄바꿈이 의미를 가진다.
- **생성 도중 에러.** 로드는 됐지만 generate 중 예외(메모리 부족 등)가 날 수 있다. 그동안 쌓인 부분 텍스트를 버리지 말고 보존한 뒤 `error`를 보낸다. UI는 부분 답변 + 에러 표식을 함께 보여 사용자가 맥락을 잃지 않게 한다. (assignment의 "스트림 중단 시 부분 텍스트 유지"와 동일 원칙.)
- **메시지 id 매칭.** 한 세션에서 빠르게 두 번 보내면 토큰이 섞일 수 있다. 모든 `token`/`done`/`aborted`에 생성 `id`를 달고, 메인은 현재 활성 `id`의 토큰만 누적한다. (assignment의 깨진/뒤섞인 SSE data 방어와 같은 발상.)

## 3. 취소(중단) 타이밍
- 브라우저 생성은 비동기 루프라 즉시 멈출 수 없다 — `stopping_criteria`(또는 플래그를 검사하는 커스텀 criteria)로 **다음 토큰 경계**에서 멈춘다.
- **낙관적 UI.** 메인은 `abort`를 보낸 즉시 스트리밍 표시를 멈추고 입력을 다시 활성화한다. 실제 Worker의 `aborted` 응답은 뒤늦게 와도 된다(이미 멈춘 상태와 합류).
- **취소 후 잔여 토큰.** abort와 거의 동시에 도착한 토큰은 무시한다(활성 id가 아니면 드롭). 그렇지 않으면 멈춘 뒤에 글자가 더 찍힌다.
- **로드 중 취소.** 다운로드 중 취소는 의미가 모호하다 — 기본은 "다운로드는 계속(캐시 이득), UI만 idle로". 명세에 따라 조정.

## 4. 런타임 가용성(WebGPU/WASM)
- WebGPU 감지: `"gpu" in navigator && await navigator.gpu.requestAdapter()`가 null 아님. 실패하면 WASM 폴백.
- 폴백을 **조용히** 하지 말고, 느릴 수 있음을 한 번 안내(WASM은 WebGPU보다 수 배 느림).
- 구형/비대응 브라우저(또는 모바일 메모리 부족)에서는 모델 로드 자체가 실패할 수 있다 → "이 브라우저에서는 AI 기능을 쓸 수 없어요" 안내 + 기능 버튼 비활성. 절대 무한 스피너로 방치하지 않는다.

## 5. 상태 누락 → 멈춘 UI
표현해야 하는 상태를 하나라도 빼면 사용자는 "고장났다"고 느낀다. 최소 상태 머신:
```
idle ──(load)──► downloading ──(ready)──► ready ──(generate)──► generating ──(done/aborted)──► ready
  ▲                  │                        │                      │
  └───(error)────────┴────────────────────────┴──────────────────────┘   error → 재시도 경로 제공
```
- `downloading`엔 항상 퍼센트(또는 받은 MB)를 같이.
- `generating`엔 취소 버튼을 노출.
- `error`엔 사람이 읽을 메시지 + 재시도 액션.

## 6. 메모리·성능
- 모델은 수백 MB~1GB+ RAM을 점유한다. 같은 탭에서 모델을 두 개 동시에 들지 않는다.
- 매우 긴 대화는 컨텍스트 한계를 넘는다 — 최근 N턴만 프롬프트에 넣거나(슬라이딩 윈도우) 토큰 길이로 자른다.
- `max_new_tokens`를 합리적으로 제한(기본 512). 무한 생성으로 탭이 얼지 않게.
- Worker는 한 번 만들어 재사용한다(매 요청 새 Worker 생성 = 매번 모델 재로드).

## 7. Next.js/Worker 빌드 함정
- Worker는 클라이언트 전용 — 엔진을 쓰는 컴포넌트/훅은 `"use client"`, Worker 생성은 마운트 이후(`useEffect`)로. SSR에서 `Worker`/`navigator.gpu` 접근 금지.
- 먼저 **기본 설정으로 빌드/실행**하고, 깨질 때만 최소 설정 추가:
  - Node 전용 모듈을 번들이 끌어오면 `next.config.mjs`의 `serverExternalPackages` 또는 webpack `resolve.fallback`로 차단.
  - `.wasm`/`.onnx` 자산 404면 `transformers` env(`env.allowLocalModels`, `env.backends.onnx.wasm.wasmPaths`)로 경로를 CDN/`/`에 맞춘다.
- 멀티스레드 WASM(SharedArrayBuffer)을 *명시적으로* 켤 때만 COOP/COEP 응답 헤더 필요. 기본(WebGPU/단일스레드)엔 넣지 않는다 — 잘못 넣으면 외부 리소스가 막힌다.

## 8. UX 디테일(자동 스크롤·첫 토큰)
- **자동 스크롤.** 토큰이 쌓이는 동안 하단 추종. 단 사용자가 위로 스크롤하면 추종 중단(강제 스크롤은 거슬린다). assignment의 "자동 스크롤 + 사용자 개입 시 해제" 교훈.
- **첫 토큰 지연.** 모델 로드(수십 초)와 첫 토큰까지의 지연은 다른 단계다. `generating` 진입 후 첫 `token` 전까지 "생각 중…" 표시로 빈 화면을 피한다.
- **입력 잠금.** `generating` 중 새 전송을 막거나 큐잉. 동시에 두 생성이 같은 세션에 쓰면 토큰이 섞인다(§2 id 매칭과 연계).

## 9. assignment 교훈 매핑표 (개념만 차용, 코드 복사 금지)
| assignment(SSE/서버) | builbook(Worker/온디바이스) |
|----------------------|------------------------------|
| `text_delta` 이벤트 | `token` postMessage |
| `done` 이벤트 → `es.close()` | `done` postMessage → Worker 유지(재사용) |
| SSE data JSON 깨짐 방어 | 토큰 id 매칭 + 활성 id만 누적 |
| 스트림 중단 시 부분 텍스트 유지 | generate 예외 시 부분 텍스트 보존 + `error` |
| 에러 shape 분기(400 `{error}`/500 `{message}`) | 로드 실패 vs 생성 실패 vs 미지원 분기 |
| 페이지네이션 limit+1 | (해당 없음 — 로컬 대화는 IndexedDB 일괄) |
| 자동 스크롤 + 사용자 개입 해제 | 동일 |
| 명시적 트리거(자동 스트림 제거) | "기능 사용" 게이팅(자동 다운로드 금지) |
